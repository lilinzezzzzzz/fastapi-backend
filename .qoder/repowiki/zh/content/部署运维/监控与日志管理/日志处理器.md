## 简介

本项目采用基于 Loguru 的现代化日志处理方案，提供了灵活、高性能的日志管理能力。日志处理器支持多种输出格式（文本和JSON）、动态命名空间、自动轮转和保留策略，以及完整的追踪ID集成。

**主要特性包括**：
- **多格式支持**：文本格式和JSON Lines格式
- **动态命名空间**：支持按业务模块创建独立的日志命名空间
- **智能轮转**：基于时间、大小的灵活轮转策略
- **追踪集成**：与分布式追踪系统无缝集成
- **多输出源**：同时支持控制台和文件输出
- **完整时区支持**：支持UTC和本地时区，严格的一致性检查
- **ISO 8601格式化**：标准化的时间格式输出
- **灵活的目录组织**：支持子目录和扁平目录结构
- **增强的追踪ID验证**：使用 is_valid_trace_id 函数进行严格验证，支持 'unknown' 和 '-' 值处理
- **维护性改进**：移除了不必要的属性赋值，提升了代码质量
- **配置系统重构**：支持.env文件加载、环境检测和配置验证
- **启动日志记录**：提供详细的启动阶段日志记录和错误报告

## 核心组件

### LoggerHandler 主处理器

LoggerHandler是日志系统的核心控制器，负责配置和管理所有日志输出。

**主要功能**：
- 配置日志级别、输出格式、轮转策略
- 管理多个命名空间的日志输出
- 提供动态日志记录器创建
- **新增**：完整的时区支持和规范化处理
- **新增**：ISO 8601时间格式化
- **更新**：灵活的目录组织和命名空间管理
- **更新**：增强的追踪ID验证机制
- **维护性改进**：移除了不必要的 _is_utc 属性赋值，提升了代码质量和可维护性

**关键配置参数**：
- `level`：日志级别（INFO、DEBUG等）
- `base_log_dir`：日志根目录
- `use_subdir`：是否使用子目录分隔日志，True则按log_namespace创建子目录，False则所有日志存放在base_log_dir下
- `rotation`：轮转策略（时间或大小）
- `retention`：保留策略（天数）
- `log_format`：输出格式（TEXT或JSON）
- **新增**：`timezone`：日志时区，支持字符串、ZoneInfo对象或datetime.timezone
- **新增**：`enqueue`：多进程安全队列写入

**更新**：重构的命名空间系统
- 默认命名空间从 'system' 更改为 'default'
- 支持灵活的目录组织策略
- 新增 `_get_log_dir` 方法处理目录解析

**更新**：增强的追踪ID验证机制
- 使用 `context.is_valid_trace_id()` 替代简单的 None 检查
- 支持对 'unknown' 和 '-' 值的严格验证
- 提供更好的错误报告和调试信息

**维护性改进**：移除不必要的属性赋值
- 移除了 LoggerHandler.__init__ 中的 `_is_utc` 属性赋值
- 这是一个维护性改进，不影响外部功能
- 提升了代码的简洁性和可维护性

### 追踪ID验证系统

**更新**：增强的追踪ID验证机制

系统现在使用 `is_valid_trace_id` 函数进行更严格的追踪ID验证：

**验证规则**：
- 必须是字符串类型
- 不能是 'unknown' 值
- 不能是 '-' 值
- 支持 None 值回退到上下文获取

**验证流程**：
```mermaid
flowchart TD
Start([获取 trace_id]) --> CheckType{检查类型}
CheckType --> |不是字符串| Invalid[返回无效]
CheckType --> |是字符串| CheckValue{检查特殊值}
CheckValue --> |是 'unknown'| Invalid
CheckValue --> |是 '-'| Invalid
CheckValue --> |其他| Valid[返回有效]
Invalid --> TryContext[尝试从上下文获取]
TryContext --> HasContext{上下文有值?}
HasContext --> |是| ReturnContext[返回上下文值]
HasContext --> |否| Default[返回 '-']
ReturnContext --> End([结束])
Default --> End
Valid --> End
```
