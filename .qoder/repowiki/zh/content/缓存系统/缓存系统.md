### Redis 客户端 RedisClient
**更新** 从pkg/toolkit/cache.py重构为pkg/toolkit/redis_client.py，提供更清晰的Redis客户端封装

- 能力概览
  - 键值：set_value/get_value
  - 结构体：set_dict/get_dict、set_list/get_list_value、push_to_list/get_list/left_pop_list
  - 哈希：set_hash/get_hash
  - TTL：set_expiry/key_exists/get_ttl
  - 分布式锁：acquire_lock/release_lock
  - 批量：batch_delete_keys、remove_from_list
- 设计要点
  - 通过 session_provider 获取 Redis[str] 会话，确保字符串专用的类型安全性
  - 统一异常包装为 RedisOperationError，便于上层捕获与降级
  - 使用 orjson 进行高性能序列化/反序列化
  - 支持异步上下文管理器模式，自动处理连接生命周期
  - 通过 `decode_responses=True` 配置，返回字符串而非字节，改善开发者体验
- 复杂度与性能
  - 基本操作为 O(1)，列表操作为 O(n)（n 为列表长度）
  - 分布式锁采用 SET NX PX 原语，避免 Lua 脚本开销

### 分布式锁与并发控制
- 获取锁
  - 使用 SET key identifier NX PX expire_ms 原语，避免脚本开销
  - 循环重试，支持超时与间隔
- 释放锁
  - 使用 Lua 脚本判断锁值一致性后再删除，避免误删他人锁
- 使用建议
  - 为不同资源设定独立的锁键，必要时加前缀区分
  - 锁过期时间应短于业务执行时间，避免长时间阻塞

## 性能考量
- 连接与序列化
  - 使用连接池与 LazyProxy，避免重复初始化
  - 使用 orjson 进行高性能序列化/反序列化
  - 通过 `decode_responses=True` 配置，返回字符串而非字节，减少额外解码开销
- 操作选择
  - 读多写少场景优先使用 GET/LRANGE/HGET 等 O(1)/O(n) 命令
  - 列表操作注意 n 值，必要时分页或限制长度
- 锁与并发
  - 使用原语 SET NX PX，减少脚本开销
  - 合理设置锁过期时间，避免长时间阻塞
- 监控与告警
  - 建议埋点统计 Redis 命中率、慢查询与异常次数
  - 结合应用日志与指标系统进行告警

## 故障排查指南
- 常见问题
  - Redis 未初始化：调用前检查 lifespan 是否执行
  - 键不存在：确认键命名与过期策略
  - 反序列化失败：检查存储值是否为有效 JSON
  - 分布式锁误释放：确保 identifier 一致
- 定位步骤
  - 查看应用日志中 Redis 初始化与异常记录
  - 在中间件与认证流程中增加关键路径日志
  - 使用 get_ttl/key_exists 辅助定位过期与存在性问题
- 降级策略
  - Redis 操作异常时快速失败并记录
  - 对读多写少场景可允许短暂降级读取数据库

## 结论
本项目的缓存系统以 Redis 为核心，经过重构后采用pkg/toolkit/redis_client.py提供了更清晰的客户端封装，结合统一的缓存客户端与懒加载代理，实现了认证链路中的高效令牌校验。通过合理的键命名、TTL 策略与分布式锁，系统在并发与可靠性方面具备良好基础。

**更新** 最新的类型注解改进包括：
- **Redis[str] 类型注解**：将Redis客户端类型从通用类型更新为字符串专用类型，增强类型安全性
- **decode_responses=True 配置**：在连接池和Redis客户端中都设置了响应解码，改善开发者体验

建议后续补充缓存穿透与雪崩的主动防护策略（如布隆过滤器、互斥锁与随机过期），并完善监控与告警体系。
