## 简介
本文件系统化梳理并文档化本项目的加密工具包，重点覆盖基于对称加密的 AES 实现（Fernet 包装）、算法注册系统、密钥管理、初始化与懒加载、便捷函数、以及密码哈希方案。文档同时给出使用示例、安全最佳实践、常见陷阱、算法选择建议与性能注意事项，并提供兼容性与调试技巧。

**更新** 本次更新重构了加密模块架构，新增 PasswordHandler 类提供基于 bcrypt 的密码加密功能，增强了系统的密码处理能力，支持更安全的用户认证场景。新的架构采用统一接口中心和算法注册系统，提供了更好的可扩展性和维护性。

## 核心组件
- **重构后的统一接口**：pkg/crypter/__init__.py 提供完整的加密工具接口，包括算法枚举、注册系统、抽象基类和工厂函数。
- AESCipher：基于 Fernet 的对称加密实现，负责密钥解析、加密、解密与密钥生成；支持字符串与字节两种密钥输入形式。
- BaseCryptoUtil：抽象基类，约束所有加密实现必须具备的接口与最小校验（如密钥非空）。
- EncryptionAlgorithm：算法枚举，当前包含 AES；预留扩展（如 SM4）。
- register_algorithm：装饰器，将具体算法实现注册到全局注册表。
- get_crypto_class：根据算法枚举返回对应实现类，便于业务层按需获取。
- init_aes_cipher / aes_cipher：应用级初始化与懒加载代理，避免模块导入时未初始化问题。
- Hasher：基于 bcrypt 的密码哈希与校验工具，支持自定义迭代轮数，异步接口通过线程池执行同步逻辑。
- **新增** PasswordHandler：基于 bcrypt 的静态密码加密与验证工具，提供简洁的密码处理接口。

### AESCipher 组件
- 设计要点
  - 基于 Fernet 的对称加密，内部自动处理随机 IV 与完整性校验（HMAC），保证机密性与抗篡改。
  - 支持字符串与字节两种密钥输入，自动进行编码转换。
  - 提供静态方法生成符合要求的密钥（url-safe base64 编码，长度满足 Fernet 要求）。
  - 便捷函数 aes_encrypt/aes_decrypt/aes_generate_key，便于快速使用。
- 错误处理
  - 非法密钥或空密钥抛出明确错误信息，提示密钥格式与生成方式。
  - 解密失败（令牌无效/密钥错误）抛出明确错误，便于上层区分。
- 数据流
  - 明文字符串经 UTF-8 编码后加密，返回 UTF-8 编码的密文字符串；解密流程相反。
  - 空字符串输入返回空字符串，避免额外分支。

### 算法注册与工厂系统
**更新** 新增的算法注册系统提供了更好的可扩展性：

- **register_algorithm 装饰器**：将具体算法类注册到全局注册表，确保 get_crypto_class 可按枚举返回正确实现。
- **get_crypto_class 工厂函数**：根据算法枚举返回对应类，若未注册则抛出未实现错误，便于扩展新算法。
- **自动注册机制**：通过模块导入触发具体实现的注册，无需手动维护注册表。

### 密码哈希（bcrypt）增强
**更新** 项目现已提供双重密码处理方案：

#### Hasher 类（异步接口）
- Hasher：封装 bcrypt 的盐生成与哈希计算，支持自定义迭代轮数；提供异步接口，内部通过线程池执行同步逻辑。
- 接口约定：空明文或空哈希返回 False；异常输入统一返回 False，避免泄露细节。

#### PasswordHandler 类（静态接口）
**新增** PasswordHandler：提供静态方法的密码处理工具，无需实例化即可使用。

- **功能特性**：自动加盐、高计算成本、不可逆性，支持密码验证和重新加密检查。
- **错误处理**：捕获无效格式和编码错误，记录日志并返回 False。
- **使用优势**：静态方法设计，避免实例化开销，适合频繁调用的场景。

## 性能与安全考量
**更新** 性能与安全考量已更新以反映新的架构：

- **性能**
  - AESCipher 基于 Fernet，内部使用高性能加密库；对小文本加解密开销主要来自字符串编解码与完整性校验。
  - 懒加载代理避免应用启动时不必要的初始化；建议在应用启动阶段集中完成 init_aes_cipher。
  - bcrypt 轮数越高安全性越好但计算成本越高，建议结合业务场景权衡（默认轮数可满足大多数场景）。
  - **新增** PasswordHandler 采用静态方法设计，避免实例化开销，适合频繁调用的场景。
  - **新增** 算法注册系统采用装饰器模式，注册过程在模块导入时自动完成，避免运行时开销。
- **安全**
  - 密钥必须是 32 字节 URL 安全 Base64 编码，且妥善保管；建议使用 aes_generate_key 生成并在安全渠道分发与存储。
  - 同一明文多次加密得到的密文不同（由随机 IV 与 HMAC 保证），避免统计分析。
  - 解密失败严格区分"令牌无效/密钥错误"与"篡改"，便于定位问题。
  - 环境变量中不应硬编码密钥，应通过安全配置源注入（参考环境示例文件）。
  - **新增** PasswordHandler 自动处理盐生成和错误处理，提供更安全的密码存储方案。
  - **新增** 统一接口中心提供更好的安全边界控制，所有加密操作都经过标准化的验证流程。

## 故障排查指南
**更新** 故障排查指南已更新以反映新的架构和新增功能：

- **"密钥无效"或"空密钥"**
  - 现象：初始化 AESCipher 抛出错误。
  - 处理：确认密钥为 32 字节 URL 安全 Base64 编码；可通过 aes_generate_key 生成新密钥并替换。
- **"解密失败：令牌无效或密钥错误"**
  - 现象：解密抛出错误。
  - 处理：确认使用的是加密时的同一密钥；检查密文是否被篡改；核对编码一致性（UTF-8）。
- **"未初始化 AES Cipher"**
  - 现象：访问 aes_cipher 抛出运行时错误。
  - 处理：在应用启动阶段调用 init_aes_cipher(secret_key) 完成初始化。
- **"空字符串输入"**
  - 现象：加密/解密空字符串返回空字符串。
  - 处理：业务侧需显式处理空值分支，避免误判。
- **"算法未注册"**
  - 现象：调用 get_crypto_class 抛出未实现错误。
  - 处理：确认算法已在模块导入时注册；检查 @register_algorithm 装饰器使用是否正确。
- **"密码哈希失败"**
  - 现象：PasswordHandler 验证密码时返回 False。
  - 处理：检查密码格式和编码；确认使用正确的哈希值；查看日志中的错误信息。
- **"bcrypt 轮数不匹配"**
  - 现象：哈希验证失败或需要重新加密。
  - 处理：使用 PasswordHandler.needs_rehash 检查是否需要重新加密；调整轮数配置。

- **完整性与抗篡改**
  - 依赖 Fernet 的 HMAC 校验；避免对密文进行二次修改。
- **性能与可用性**
  - 合理设置 bcrypt 轮数；在应用启动阶段完成 AES 初始化；避免在热路径重复构造实例。
- **新增** **算法扩展**
  - 使用 register_algorithm 装饰器注册新算法；遵循 BaseCryptoUtil 接口规范。
- **新增** **密码处理**
  - 使用 PasswordHandler 进行密码加密和验证；避免直接使用底层 bcrypt 库。
  - 定期检查密码哈希是否需要重新加密；根据硬件性能调整轮数。

### 算法选择与兼容性
**更新** 算法选择与兼容性已更新：

- **AES（Fernet）**
  - 特点：内置完整性校验、随机 IV、易用性强；适合大多数对称加密场景。
  - 兼容性：依赖 cryptography 库；确保部署环境安装相应二进制依赖。
- **bcrypt**
  - 特点：专为密码设计，自适应盐与可调轮数；适合密码存储与校验。
  - 兼容性：Python bcrypt 库；注意轮数变化导致的哈希格式差异。
  - **新增** PasswordHandler：提供静态接口，简化 bcrypt 使用；自动处理错误和日志记录。
- **新增** **算法注册系统**
  - 特点：装饰器驱动的注册机制；自动导入触发注册；支持未来扩展。
  - 兼容性：基于 Python 3.8+ 的类型注解；与现有代码完全兼容。
